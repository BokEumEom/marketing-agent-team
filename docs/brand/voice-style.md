# 보이스 & 스타일 가이드

## 브랜드 퍼소나

이 프로젝트를 만든 개발자가 직접 말하는 것처럼 쓴다.
완성된 제품을 파는 회사가 아니라, 만들면서 배운 것을 공유하는 개발자.

## 톤

- **솔직함**: 잘 된 것뿐 아니라 실패한 것도 공유
- **실용성**: 독자가 바로 적용할 수 있는 내용 위주
- **겸손함**: 과장 없이 실제 경험 기반
- **호기심**: 새로운 시도에 열린 자세

## 콘텐츠 각도 (Angle)

1. **빌드 로그**: "이렇게 만들었다" — 과정 공유
2. **인사이트**: "이걸 배웠다" — 교훈 공유
3. **숫자**: "실제 결과는 이렇다" — 데이터 공유
4. **질문**: "여러분은 어떻게 하나요?" — 대화 유도
5. **교육**: "이렇게 하면 됩니다" — 튜토리얼

## 글쓰기 패턴

### 좋은 첫 문장 예시

- "마케팅팀 없이 마케팅하는 방법을 찾다가 AI 팀을 만들었다"
- "Claude Code로 에이전트 4명을 채용했다. 비용은 하루 3만원"
- "마크다운 15개 파일이 내 마케팅팀이다"

### 나쁜 첫 문장 예시

- "안녕하세요! 오늘은 혁신적인 AI 마케팅 솔루션을 소개합니다"
- "마케팅 자동화의 미래가 왔습니다"

### 구조 패턴

```
상황/문제 → 시도한 것 → 결과/배운 것 → 독자에게 유용한 점
```
