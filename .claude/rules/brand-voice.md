# 브랜드 보이스 규칙

## 핵심 톤앤매너

**전문적이되 친근하게.** 개발자/마케터 동료에게 말하듯이 쓴다.

- 딱딱한 기업 언어 X
- 지나친 구어체 X
- 실용적이고 솔직하게 O

## 언어 규칙

- **한국어 기본**, 기술 용어는 영어 그대로 사용 (예: Claude Code, MCP, cron)
- 존댓말 사용 (-습니다/-입니다 스타일보다 -요/-어요 스타일 선호)
- 이모지: 포스트당 최대 2개, 의미 있을 때만 사용

## 콘텐츠 원칙

1. **구체적 숫자와 사례** — "마케팅 자동화" 대신 "하루 $30로 콘텐츠 5개 자동 발행"
2. **내부 시각 공유** — 어떻게 만들었는지, 무엇이 실패했는지 솔직하게
3. **실용적 인사이트** — 읽은 사람이 바로 써먹을 수 있는 내용
4. **자랑보다 교육** — 제품 홍보보다 지식 공유로 신뢰 쌓기

## 피해야 할 표현

- "혁신적인", "패러다임 전환", "게임체인저" — 진부한 마케팅 언어
- "최고의", "유일한", "완벽한" — 근거 없는 최상급
- 과도한 느낌표(!) — 1개 포스트에 1개 이하
- 영어 직역체 — "활용하다" 대신 "쓰다", "구현하다" 대신 "만들다"

## 포스트 형식별 가이드

### X/Twitter
- 240자 이내
- 첫 문장이 후킹 (질문, 숫자, 반전)
- 스레드는 최대 5개 트윗

### LinkedIn
- 첫 2줄에 핵심 메시지
- 단락 짧게 (3줄 이하)
- 해시태그 3-5개, 마지막에 배치

### Threads
- X보다 조금 더 캐주얼
- 대화 유도 질문으로 마무리

### Maily 뉴스레터 — Paul Graham 스타일

**목표:** 읽고 나서 생각이 바뀌는 글. 정보 전달이 아니라 관점 전환.

**구조 원칙:**
- 헤더(##) 사용 금지 — 산문으로 흐른다
- 단락 = 하나의 생각. 2-4문장이면 충분
- 빈 줄로 단락을 크게 분리해 숨 고르는 공간 만들기
- 결론을 앞에 쓰지 않는다 — 읽다 보면 자연히 도달하게

**시작 방법:**
- 구체적 장면 또는 사건으로 시작 (주장/테제 X)
- "나는 어느 날 ~을 발견했다" 형식이 이상적
- 첫 두 문장 안에 궁금증이 생겨야 한다

**문장 원칙:**
- 모든 문장이 다음 문장을 당겨야 한다
- 군더더기 단어 제거: "사실", "결국", "즉", "그래서" 최소화
- 짧은 문장 뒤에 긴 문장, 또는 그 반대로 리듬 만들기
- 추상 명사보다 동사 중심으로

**마무리:**
- 질문으로 끝내거나, 역설로 끝내거나, 다음 생각의 씨앗을 심는다
- "이렇게 하세요" 같은 처방전으로 끝내지 않는다

**분량:** 600-1000자. 짧을수록 좋다. 길면 덜어낸다.

**자가 검토 질문:**
- 이 단락이 없어도 글이 통하는가? → 없애라
- 이 문장을 더 짧게 쓸 수 있는가? → 짧게 써라
- 독자가 이 문장에서 멈추고 생각할까? → 그래야 좋은 문장이다
